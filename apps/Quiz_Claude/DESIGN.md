# DESIGN.md — パステルキャンディ 🍬

> このファイルは [designmd.ai](https://designmd.ai/) 形式のデザインシステムです。
> **AIコーディングツールがこの1枚を読んで、一貫したUIを組み立てる**ことを目的にしています。

## 差し替え方法（重要）

別の見た目にしたいときは、designmd.ai から好みの `DESIGN.md` を取得し、
このファイルを置き換えたうえで、下の **トークン対応表** に従って
`src/main/resources/static/css/tokens.css` の CSS 変数を書き換えてください。
`style.css` はすべて `var(--…)` 経由で色・角丸・影を参照しているため、
`tokens.css` を差し替えるだけでアプリ全体の見た目が入れ替わります。

## コンセプト

やさしくて甘い、キャンディショップのようなUI。丸みとぷっくりした影で「押したくなる」かわいさを出す。
子どもが**ぱっと見て「たのしそう！」**と感じるよう、虹色のタイトル、背景をふわふわ浮かぶおかし、
弾むカード、正解時の紙吹雪など、動きと彩りを積極的に添える（`prefers-reduced-motion` では控えめに）。

## カラーパレット

- **背景 `#FFF5FA`**: ほんのりピンクの生成り。全画面のベース。
- **メイン `#FF9EC4` / 濃色 `#FF6FA5`**: ボタン・見出し・アクセントの主役。
- **ミント `#9EE6D0`**: 差し色。○ボタンや装飾に。
- **レモン `#FFE28A`**: サブの差し色。バッジや強調に。
- **文字 `#5A4A54`**: 黒すぎない、やわらかいブラウングレー。
- **正解 `#5BC8A0` / 不正解 `#FF8FA3`**: 判定表示の色。

## 形・質感

- 角丸は大きめ（カード 28px / ボタン 18px 以上）。
- 影は「下方向のソリッド影＋やわらかいドロップシャドウ」の二段で、ぷっくり立体感を出す。
- ボタンはホバーで少し浮き、押下で沈む（`translateY`）。
- 罫線は極力使わず、面と影で区切る。

## タイポグラフィ

- 丸ゴシック系（例: "Hiragino Maru Gothic ProN", "Rounded Mplus 1c", system-ui）。
- 見出しは太く大きく、文字間は少しゆったり。
- ゲームの「正解！」「残念・・・。」は画面を占有するほど大きく（clamp で 3rem〜7rem）。

## コンポーネント指針

- **カード**: 白面 + 大角丸 + ぷっくり影。主要コンテンツはカードに載せる。
- **ボタン（主）**: メインカラー背景・白文字・大角丸・ソリッド影。
- **ボタン（副）**: 白背景・メイン文字・淡い枠。
- **○/×ボタン**: 大きな円形。○はミント寄り、×はピンク寄り。
- **バッジ**: 小さめ角丸ピル。○＝ミント / ×＝ピンク。
- **メッセージ（Flash）**: 上部に淡い面でふわっと表示。

## アクセシビリティ

- 文字色と背景のコントラストを確保（本文はブラウングレーで黒に近い可読性）。
- ボタンはテキストラベルを持ち、色だけに依存しない（○/×は記号＋色）。
- フォーカスリングを残す。
